# 작업명

목록 추가 탭을 발송 현황 대시보드로 변경

## 목표

* `1. 목록 추가` 탭 명칭을 `1. 발송 현황`으로 변경한다.
* 기존 업로드 상세 결과 영역을 Supabase 기반 발송 현황 요약 영역으로 변경한다.
* 발송 현황 요약은 다음 3개 항목을 표시한다.
  * 차단된 메일
  * 총 수신자
  * 발송된 수신자
* 기존 `4. 작업` 그룹에서 `로그 보기` 버튼을 제거한다.
* 기존 `4. 작업` 그룹에 `전체 발송현황 초기화` 버튼을 추가한다.
* 프로그램 실행 아이콘은 `bin/nara-email.ico` 기준으로 설정한다.

## 배경

* 현재 `목록 추가` 탭은 Excel 업로드 처리 결과를 중심으로 구성되어 있다.
* 실제 운영에서는 업로드 직후 결과보다 현재 DB 기준 발송 현황을 빠르게 확인하는 화면이 필요하다.
* `Recipients`, `SendHistory`, `BlockedEmails` 테이블이 이미 Supabase에 있으므로, 첫 번째 탭을 단순 업로드 결과 화면에서 운영 현황 대시보드 성격으로 바꾼다.
* 이 프로그램은 대상 파악과 발송 실행을 위한 도구이며, 장기 이력 관리는 현재 범위에서 제외한다.
* 따라서 `전체 발송현황 초기화`는 기존 발송 이력인 `SendHistory` 전체 삭제로 정의한다.

## 관련 파일

* `MailSender_v2/MainForm.cs`
* `MailSender_v2/Data/SupabaseRestClient.cs`
* `MailSender_v2/Data/SupabaseRecipientRepository.cs`
* `MailSender_v2/Database/supabase_schema.sql`
* `MailSender_v2/MailSender_v2.csproj`
* `MailSender_v2/nara-email.ico`
* `CurrentWork.md`

## 변경 금지

* `Recipients`의 수신자 원본 데이터를 초기화 기능으로 삭제하지 않는다.
* `BlockedEmails`의 차단 목록을 초기화 기능으로 삭제하지 않는다.
* `UploadHistory`의 업로드 이력을 초기화 기능으로 삭제하지 않는다.
* 기존 Excel 업로드 기능 자체를 제거하지 않는다.
* 실제 운영 이메일 주소나 민감 설정값을 문서/로그에 남기지 않는다.

## 요구사항

### 1. 탭 명칭 변경

* 첫 번째 탭 제목을 `1. 목록 추가 (입찰공고 파일 업로드)`에서 `1. 발송 현황`으로 변경한다.
* Excel 업로드 기능은 유지하되, 탭의 주 목적은 발송 현황 확인으로 보이도록 UI 문구를 정리한다.

### 2. 발송 현황 요약 표시

* 기존 `2. 업로드 상세 결과` 그룹은 발송 현황 요약 그룹으로 변경한다.
* 표시 항목은 다음 3개로 제한한다.

|항목|기준 테이블|집계 기준|
|-|-|-|
|차단된 메일|`BlockedEmails`|전체 row 수|
|총 수신자|`Recipients`|전체 row 수|
|발송된 수신자|`SendHistory`|발송 완료로 인정되는 고유 이메일 수|

* `발송된 수신자`의 완료 인정 상태는 우선 다음 값을 기준으로 한다.
  * `Sent`
  * `ManuallyMarkedSent`
* 같은 이메일에 여러 발송 이력이 있어도 1명으로 집계한다.
* Supabase 설정이 없거나 조회 실패 시 `-` 또는 `0`으로 표시하고 로그에 원인만 남긴다.

### 3. 작업 그룹 변경

* `4. 작업` 그룹의 `로그 보기` 버튼을 제거한다.
* `4. 작업` 그룹에 `전체 발송현황 초기화` 버튼을 추가한다.
* 초기화 버튼은 파괴적 작업이므로 클릭 시 확인 MessageBox를 반드시 표시한다.
* 초기화 작업 완료 후에는 사용자가 직접 다시 조회할 수 있도록 한다.

### 4. 전체 발송현황 초기화

* 의미: 모든 발송 이력을 삭제한다.
* 효과: `이미 발송된 이메일 제외` 조건이 초기 상태처럼 동작한다.
* 삭제 대상: `SendHistory` 전체 row
* 삭제 제외:
  * `Recipients`
  * `BlockedEmails`
  * `UploadHistory`
* 버튼 클릭 시 강한 확인 MessageBox를 표시한다.
* 확인 문구에는 `발송 이력 전체 삭제`, `복구 불가`, `수신자/차단 목록은 유지`를 포함한다.
* 사용자가 확인하면 Supabase REST API를 통해 `SendHistory` 전체 삭제를 수행한다.
* 삭제 완료 후 발송 현황 요약을 다시 조회하여 반영한다.

### 5. Supabase delete 권한과 RLS

* `supabase_schema.sql`에서 `SendHistory`에 anon `delete` 권한을 추가한다.
* 초기 테스트 정책 기준으로 `SendHistory` 전체 삭제가 가능하도록 delete RLS policy를 추가한다.
* `service_role` key는 계속 사용하지 않는다.
* delete 권한은 `SendHistory`에만 추가한다.
* `Recipients`, `BlockedEmails`, `UploadHistory`에는 초기화 목적의 delete 권한을 추가하지 않는다.

### 6. 아이콘 설정

* 프로그램 실행 아이콘은 `MailSender_v2/nara-email.ico` 또는 빌드 출력의 `bin/nara-email.ico`를 기준으로 확인한다.
* `.csproj`의 `ApplicationIcon` 설정과 실제 파일 포함 여부를 점검한다.
* 이미 설정되어 있으면 변경하지 않고 결과에 확인만 남긴다.

## 완료 조건

* 첫 번째 탭 명칭이 `1. 발송 현황`으로 변경된다.
* 발송 현황 요약 영역에 차단된 메일, 총 수신자, 발송된 수신자가 표시된다.
* `로그 보기` 버튼이 `4. 작업` 그룹에서 제거된다.
* `전체 발송현황 초기화` 버튼이 추가된다.
* `전체 발송현황 초기화` 버튼은 `SendHistory` 전체 삭제를 수행한다.
* `supabase_schema.sql`에 `SendHistory` anon delete 권한과 RLS policy가 추가된다.
* 프로그램 실행 아이콘 설정 상태가 확인된다.
* 빌드가 성공한다.

## 테스트 방법

* .NET Framework용 MSBuild로 빌드한다.
* 앱 실행 후 첫 번째 탭 명칭이 `1. 발송 현황`인지 확인한다.
* Supabase 설정이 있는 상태에서 차단된 메일, 총 수신자, 발송된 수신자 집계가 표시되는지 확인한다.
* Supabase 설정이 없는 상태에서 앱이 오류 없이 시작되는지 확인한다.
* `4. 작업` 그룹에서 `로그 보기` 버튼이 제거되었는지 확인한다.
* `전체 발송현황 초기화` 버튼 클릭 시 확인 창이 표시되는지 확인한다.
* `전체 발송현황 초기화` 실행 후 `SendHistory`가 비워지고, 발송된 수신자 집계가 0으로 표시되는지 확인한다.
* 초기화 후 `Recipients`, `BlockedEmails`, `UploadHistory` 데이터가 유지되는지 확인한다.
* 실행 파일 아이콘이 `nara-email.ico`로 표시되는지 확인한다.

## 결과

* `전체 발송현황 초기화` 기준을 `SendHistory` 전체 삭제로 확정했다.
* Supabase anon 권한에 `SendHistory` delete를 포함하고, delete RLS policy를 추가하는 방향으로 요구사항을 수정했다.
* `Recipients`, `BlockedEmails`, `UploadHistory`는 초기화 대상에서 제외하는 것으로 명시했다.
* 첫 번째 탭 명칭을 `1. 발송 현황`으로 변경했다.
* 기존 `2. 업로드 상세 결과` 표를 `2. 발송 현황 요약`으로 바꾸고, `차단된 메일`, `총 수신자`, `발송된 수신자` 3개 항목만 표시하도록 했다.
* 앱 시작 및 Excel 업로드 후 Supabase 기준 발송 현황 요약을 조회하도록 했다.
* `4. 작업` 그룹에서 `로그 보기` 버튼을 제거하고 `전체 발송현황 초기화` 버튼을 추가했다.
* `전체 발송현황 초기화`는 확인 MessageBox 후 `SendHistory` 전체 row를 삭제하고, 완료 후 발송 현황 요약을 다시 조회하도록 했다.
* `SupabaseRestClient`에 REST count 조회와 delete 호출을 추가했다.
* `SupabaseRecipientRepository`에 대시보드 요약 조회와 `SendHistory` 전체 삭제 메서드를 추가했다.
* `supabase_schema.sql`에 `SendHistory` anon delete 권한과 delete RLS policy를 추가했다.
* 프로그램 실행 아이콘은 `MailSender_v2.csproj`의 `ApplicationIcon`과 Content 항목이 이미 `nara-email.ico`로 설정되어 있음을 확인했다.
