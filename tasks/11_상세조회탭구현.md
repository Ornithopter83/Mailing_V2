# 작업명

상세 조회 탭 구현

## 목표

* `3. 상세 조회` 탭을 추가한다.
* 상세 조회 탭에서 수신자 목록을 조건별로 조회하고, 조회 결과를 선택/해제할 수 있게 한다.
* 조회 결과는 화면 페이징 없이 그리드 스크롤로 표시한다.
* 조회 결과를 Excel 파일로 내보낼 수 있게 한다.

## 배경

* 기존 `2. 발송 대상 / 메일 작성` 탭은 실제 발송 대상 선별과 발송 작업에 초점이 맞춰져 있다.
* 운영 중에는 전체 수신자, 미발송 대상, 발송완료 대상, 차단 대상 등을 발송 작업과 분리해 확인할 필요가 있다.
* Supabase REST 조회는 1000건 제한이 있으므로 DB 조회는 페이지네이션을 유지하되, 화면 표시는 페이지 넘김 없이 스크롤로 처리한다.
* 상세 조회 조건은 `config.json`에 저장하여 다음 실행 시 유지한다.

## 관련 파일

* `MailSender_v2/MainForm.cs`
* `MailSender_v2/Data/SupabaseRecipientRepository.cs`
* `MailSender_v2/Data/SupabaseRestClient.cs`
* `MailSender_v2/Mailing/RecipientListItem.cs`
* `MailSender_v2/Upload/UploadResultExporter.cs`
* `CurrentWork.md`

## 변경 금지

* 상세 조회 탭에서 SMTP 발송 기능을 수행하지 않는다.
* 상세 조회 탭에서 수동 발송완료 처리 기능을 수행하지 않는다.
* 기존 `1. 발송 현황`, `2. 발송 대상 / 메일 작성` 탭의 동작 의미를 바꾸지 않는다.
* DB 스키마를 변경하지 않는다.
* Supabase service role key 또는 민감 정보를 코드/문서/로그에 기록하지 않는다.
* 조회 결과 Excel 내보내기에 실제 운영상 불필요한 민감 정보를 과도하게 포함하지 않는다.

## 요구사항

### 1. 탭 구성

* 기존 탭 뒤에 `3. 상세 조회` 탭을 추가한다.
* 탭 상단에는 조회 조건 영역을 한 줄에 가깝게 배치한다.
* 조회 조건 아래 영역은 결과 그리드로 사용한다.
* 하단 또는 우측에는 선택/해제, Excel 저장 등 작업 버튼을 배치한다.

### 2. 조회 조건

* 공고일자 시작일과 종료일은 `20230101` 형식의 8자리 텍스트로 입력받는다.
* 상태 체크박스는 최소 다음 값을 제공한다.
  * 미발송
  * 발송완료
  * 차단됨
* 조회 건수는 NumericUpDown 또는 유사 컨트롤로 지정한다.
* `검색` 버튼을 눌렀을 때 조건에 맞는 목록을 조회한다.
* 마지막 조회 조건은 설정 파일에 저장한다.
* 1차 구현에서는 텍스트 키워드 검색은 포함하지 않는다.

### 3. 상태 판단 기준

* `발송완료`는 `SendHistory.Status`가 `Sent` 또는 `ManuallyMarkedSent`인 이메일로 판단한다.
* `미발송`은 발송완료 이력이 없고 차단 목록에도 없는 이메일로 판단한다.
* `차단됨`은 `BlockedEmails.NormalizedEmail`에 포함된 이메일로 판단한다.
* 여러 상태 체크박스가 선택된 경우 OR 조건으로 조회한다.
* 상태 체크박스가 모두 해제된 경우 전체 상태를 조회한다.

### 4. 결과 그리드

* 결과 그리드는 최소 다음 컬럼을 표시한다.
  * 선택
  * 이메일
  * 정규화 이메일
  * 기관명
  * 공고일자
  * 공고명
  * 담당자
  * 연락처
  * 상태
  * 최근 발송처리일
  * 차단사유
  * 차단등록일
* 사용자는 행 단위 체크박스로 선택/해제할 수 있다.
* 전체 선택과 전체 해제 버튼을 제공한다.
* 결과 그리드는 읽기 전용으로 유지한다.

### 5. 화면 표시

* 화면 페이징은 제공하지 않는다.
* 조회된 결과는 DataGridView 스크롤로 확인한다.
* 조회 건수 제한은 전체 조회 결과의 최대 건수로 적용한다.
* Supabase 1000건 제한에 걸리지 않도록 기존 페이지네이션 조회 메서드를 사용한다.
* 전체 조회 건수와 선택 건수를 표시한다.

### 6. Excel 저장

* `Excel 저장` 버튼을 제공한다.
* 저장 대화상자의 확장자는 `.xlsx`로 고정한다.
* 사용자가 파일명을 지정할 수 있게 한다.
* 기본 저장 대상은 선택된 행으로 한다.
* 전체 조회 결과 저장 옵션을 제공한다.
* 전체 조회 결과 저장 옵션을 선택하면 현재 조회 조건으로 조회된 전체 결과를 저장한다.
* Excel에는 상세 조회 결과에서 운영 확인에 필요한 컬럼을 출력하되 정규화 이메일은 제외한다.
* 저장 완료 후 MessageBox로 완료 메시지를 표시한다.

### 7. 로그와 오류 처리

* 조회 시작, 조회 완료, Excel 저장 완료는 로그에 남긴다.
* 조회 실패 또는 저장 실패 시 MessageBox와 로그로 원인을 표시한다.
* 로그에는 민감 정보 대신 건수와 상태 중심 정보를 남긴다.

## 완료 조건

* `3. 상세 조회` 탭이 표시된다.
* 공고일자 기간, 상태 체크박스, 조회 건수, 검색 버튼으로 조회할 수 있다.
* 미발송/발송완료/차단됨 상태가 기준에 맞게 표시된다.
* 결과 그리드에서 행 선택/해제와 전체 선택/해제가 가능하다.
* 결과 그리드는 페이지 넘김 없이 스크롤로 조회된다.
* 조회 결과를 `.xlsx` 파일로 저장할 수 있다.
* Excel 저장 완료 시 완료 메시지가 표시된다.
* 상세 조회 조건이 `config.json`에 저장되고 다음 실행 시 복원된다.
* 기존 발송 대상 조회와 SMTP 발송 흐름이 변하지 않는다.
* 빌드가 성공한다.

## 테스트 방법

* .NET Framework용 MSBuild로 빌드한다.
* 사용자에 의해 상세 조회 조건별 동작을 직접 확인한다.

## 검토 메모

* Excel 저장은 `선택 행만 저장`을 기본값으로 한다.
* 전체 저장 옵션도 제공한다.
* DB 조회 페이지네이션은 유지하되 화면 페이징은 제거한다.

## 결과

* `3. 상세 조회` 탭을 추가했다.
* 공고일자 시작/종료는 `yyyyMMdd` 텍스트 입력으로 구성했다.
* 미발송/발송완료/차단됨 체크박스, 조회 건수, 검색 버튼을 구성했다.
* 상세 조회 결과는 페이지 넘김 없이 그리드 스크롤로 표시한다.
* 상세 조회 결과와 Excel에는 운영 확인에 필요한 상세 컬럼을 표시한다.
* 행 단위 선택, 전체 선택, 전체 해제를 구현하고 페이지 이동 후에도 선택 상태가 유지되도록 했다.
* Excel 저장은 기본적으로 선택 행만 저장하며, `전체 조회 결과 저장` 옵션을 선택하면 전체 조회 결과를 저장하도록 했다.
* Excel 저장 완료 시 MessageBox로 완료를 안내한다.
* 상세 조회 조건은 `config.json`에 저장하고 다음 실행 시 복원한다.
* 상세 조회용 DTO와 Excel exporter를 추가했다.
* Supabase 조회는 기존 페이지네이션 조회 메서드를 사용하도록 했다.
* SMTP 발송 및 수동 발송완료 흐름은 변경하지 않았다.
* .NET Framework용 MSBuild Debug 빌드 성공을 확인했다.
