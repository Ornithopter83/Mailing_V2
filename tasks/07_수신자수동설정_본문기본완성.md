# 작업명

수신자 수동 설정 및 본문 기본 완성

## 목표

* 메일 작성 영역에 수동 수신자 입력 기능을 추가한다.
* 메일 작성 입력 순서를 `수신 -> 참조 -> 제목 -> 본문`으로 정리한다.
* `config.json` 기반으로 제목, 참조, 본문 기본값을 완성한다.
* 작성 중인 메일 설정을 저장하고 다시 불러올 수 있게 한다.
* DB 조회 대상과 수동 입력 대상의 발송 순서를 명확히 분리한다.

## 배경

* 현재 발송 대상은 Supabase에서 조회한 메일링 리스트 중심이다.
* 운영 중에는 DB 조회 대상 외에 수동으로 입력한 수신자에게 같은 메일을 보내야 하는 경우가 있다.
* 본문 기본값은 `config.json`에 두고 프로그램 실행 시 자동 반영하는 흐름으로 정리한다.
* 수동 수신자는 메일링 리스트 발송 이후 마지막에 한 번에 처리한다.

## 변경 금지

* 수동 입력 대상자를 `Recipients` 테이블에 자동 등록하지 않는다.
* 수동 입력 대상자의 발송 이력을 메일링 리스트 조회 대상과 같은 방식으로 잘못 연결하지 않는다.
* DB 조회 대상의 기존 선택/수동 발송완료 처리 의미를 변경하지 않는다.
* 실제 SMTP 발송 테스트는 별도 사용자 확인 전까지 수행하지 않는다.

## 요구사항

### 1. 메일 작성 UI 정리

* 메일 작성 그룹의 입력 순서를 다음과 같이 변경한다.

```text
1. 수신
2. 참조
3. 제목
4. 본문
5. 첨부 파일
6. 작업 버튼
```

* `수신` TextBox를 추가한다.
* 수신/참조 입력은 세미콜론(`;`)으로 여러 이메일을 구분한다.
* 수신/참조 입력값은 앞뒤 공백을 제거하여 검사한다.
* 수신/참조 이메일 형식 오류가 있으면 발송 전 사용자에게 표시한다.

### 2. config.json 반영 범위

* 프로그램 실행 시 `config.json`을 기본으로 읽는다.
* 메일 작성 화면의 기본값은 다음 설정을 사용한다.

|설정|UI 반영 위치|
|-|-|
|`DefaultTo`|수신 TextBox|
|`DefaultCc`|참조 TextBox|
|`Subject`|제목 TextBox|
|`Body`|본문 RichTextBox|
|`Images`|본문 CID 이미지|
|`Download`|첨부 파일 목록|

* `config.template.json`에도 동일한 설정 항목을 추가한다.
* 기존 `config.json`에 새 항목이 없어도 기본값으로 보정되어야 한다.
* 기본 본문은 `Body` 배열만 사용한다.

### 3. 본문 기본 완성

* 본문은 `Body` 배열을 기본값으로 사용한다.
* 줄바꿈은 저장/불러오기 후에도 유지되어야 한다.
* `Body` 배열의 빈 문자열은 빈 줄로 유지한다.
* `Images` 항목의 `ID`는 본문 내 `{ID}` 토큰과 매칭하여 CID 이미지로 삽입한다.
* `Images.FileName`과 `Download` 경로는 절대 경로 또는 실행 폴더 기준 상대 경로를 허용한다.
* 기존 `이미지 삽입` 버튼은 현재 방식처럼 첨부 목록 추가와 본문 참조 표시를 유지한다.

### 4. 저장/불러오기

* `임시저장` 버튼은 현재 작성 중인 메일 내용을 JSON 파일로 저장한다.
* 저장 형식은 `config.json`에서 메일 작성에 필요한 항목과 호환되도록 한다.
* 저장 시 `SaveFileDialog`를 사용한다.
* 저장 기본 파일명은 `MailDraft_yyMMdd-HHmm.json` 형식을 사용한다.
* `불러오기` 버튼을 추가한다.
* 불러오기는 `OpenFileDialog`를 사용한다.
* 불러온 JSON에서 다음 항목을 화면에 반영한다.
  * `DefaultTo`
  * `DefaultCc`
  * `Subject`
  * `Body`
  * `Images`
  * `Download`
  * 첨부 파일 경로 목록이 있으면 존재하는 파일만 반영

### 5. 발송 대상 기준

* 발송 대상은 다음 두 종류로 구분한다.
  * DB 조회 후 체크된 메일링 리스트 대상
  * 수신 TextBox에 수동 입력한 대상
* 두 대상 중 하나 이상이 있어야 발송할 수 있다.
* DB 조회 대상이 있으면 먼저 개별 발송한다.
* 수동 입력 대상은 DB 조회 대상 발송 및 결과 처리 이후 마지막에 한 번에 처리한다.
* 수동 입력 대상은 `Recipients.Id`가 없으므로 발송 이력의 `RecipientId`는 비워 둔다.
* 중복 이메일은 같은 발송 작업 내에서 1회만 처리한다.

### 6. 발송 간격

* DB 조회 대상 발송 간격은 기본 5초로 한다.
* `config.json`의 `SendInterval` 값이 있으면 그 값을 사용하되, 기본 템플릿은 `5000`으로 맞춘다.
* 수동 입력 대상은 마지막에 한 번에 발송하되, 내부 구현상 개별 SMTP 메시지가 필요하면 같은 간격 정책을 적용한다.

### 7. 로그와 미리보기

* 미리보기에는 다음 정보를 표시한다.
  * 수동 수신자 수
  * DB 선택 대상 수
  * 참조
  * 제목
  * 본문
  * 첨부 파일 수
  * `Images`에 매핑된 본문 이미지와 서명 이미지
* 로그에는 DB 발송과 수동 입력 발송을 구분해서 남긴다.

## 완료 조건

* 수신 TextBox가 추가되고 입력 순서가 정리된다.
* `config.json`의 `DefaultTo`, `DefaultCc`, `Subject`, `Body`가 메일 작성 화면에 반영된다.
* `Images`의 `{ID}` 토큰이 SMTP 발송 시 CID 이미지로 변환된다.
* `Download`의 파일이 첨부 목록에 반영된다.
* 임시저장/불러오기가 JSON 형식으로 동작한다.
* 수동 입력 대상과 DB 조회 대상이 함께 있을 때 발송 대상 수가 올바르게 계산된다.
* 수동 입력 이메일 형식 검사가 동작한다.
* 빌드가 성공한다.

## 테스트 방법

* .NET Framework용 MSBuild로 빌드한다.
* `config.json`에 `DefaultTo`, `DefaultCc`, `Subject`, `Body`를 입력하고 앱 실행 시 화면 반영을 확인한다.
* `Body` 배열에 빈 문자열을 넣어 빈 줄이 유지되는지 확인한다.
* `Images`에 `ID`, `FileName`, `Width`를 넣고 본문 `{ID}` 토큰이 HTML CID 이미지로 변환되는지 확인한다.
* 미리보기에서 `{ID}` 토큰 위치에 실행 폴더의 이미지 파일이 표시되는지 확인한다.
* `Download`에 실행 폴더 기준 파일명을 넣고 첨부 목록에 반영되는지 확인한다.
* 수신에 세미콜론으로 복수 이메일을 입력하고 형식 검사를 확인한다.
* DB 조회 대상 없이 수동 수신자만 있을 때 발송 전 확인까지 진행되는지 확인한다.
* DB 조회 대상과 수동 수신자가 함께 있을 때 미리보기 대상 수를 확인한다.
* 임시저장 후 불러오기 시 제목/수신/참조/본문이 복원되는지 확인한다.

## 결과

* 메일 작성 영역에 `수신` TextBox를 추가하고 입력 순서를 `수신 -> 참조 -> 제목 -> 본문 -> 첨부 파일 -> 작업 버튼`으로 정리했다.
* `config.json` 설정 항목에 `DefaultTo`를 추가하고, `DefaultTo`, `DefaultCc`, `Subject`, `Body`가 메일 작성 화면 초기값으로 반영되도록 했다.
* `config.template.json`과 README 예시를 새 설정 항목 및 기본 발송 간격 5초 기준으로 갱신했다.
* `임시저장`은 `SaveFileDialog`로 JSON 파일을 저장하도록 바꾸고, `불러오기` 버튼과 `OpenFileDialog` 기반 복원 기능을 추가했다.
* 수신/참조 이메일은 세미콜론(`;`) 기준으로 분리하고 발송 전 형식을 검사하도록 했다.
* DB 조회 대상과 수동 입력 대상의 발송 순서를 분리했다. DB 대상이 먼저 발송되고, 수동 입력 대상은 중복 제거 후 마지막에 발송된다.
* 수동 입력 대상은 `RecipientId` 없이 발송 이력에 기록되도록 했다.
* 미리보기에 수동 수신자 수, DB 선택 대상 수, 참조, 제목, 본문, 첨부 파일 수를 표시하도록 했다.
* `config.json` 본문 형식은 `Body` 배열만 사용하도록 변경했다.
* `Body` 배열의 각 항목을 줄 단위로 합쳐 RichTextBox에 반영하고, 임시저장 시에도 줄 단위 배열로 저장되도록 했다.
* `Images` 설정의 `ID`/`FileName`/`Width`를 읽고, SMTP HTML 본문에서 `{ID}` 토큰을 CID 이미지 태그로 변환하도록 했다.
* 작성 화면의 RichTextBox는 `{ID}` 토큰을 텍스트로 유지하고, `미리보기` 버튼은 별도 HTML 미리보기 창을 열어 토큰 위치에 실행 폴더의 이미지 파일을 표시하도록 했다.
* `Download` 설정의 파일을 실행 폴더 기준 상대 경로 또는 절대 경로로 찾아 첨부 목록에 자동 반영하도록 했다.
* 수동 발송완료 또는 SMTP 발송 완료 후 발송 대상 목록을 자동 재조회하지 않도록 변경했다. 이후 목록 갱신은 사용자가 `조회` 버튼을 눌러 수행한다.
* `config.template.json`과 README 예시를 `Body`, `Images`, `Download` 기준으로 갱신했다.
* `MainForm.cs`가 Visual Studio Designer 대상이 되지 않도록 `MailSender_v2.csproj`의 `SubType`을 `Code`로 유지했다.
