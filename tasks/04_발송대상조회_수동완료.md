# 작업명

발송대상조회_수동완료

## 목표

* Supabase에 저장된 대상자를 조건에 따라 조회한다.
* 선택된 대상자를 실제 메일 발송 없이 `발송완료 처리`할 수 있게 한다.

## 배경

* 기존 방식으로 이미 보낸 메일이 있으므로, Supabase DB에 발송 이력을 맞추는 기능이 필요하다.
* 수동 발송완료 처리된 대상은 이후 `이미 발송된 이메일 제외` 조건에 포함되어야 한다.

## 변경 금지

* SMTP 실제 발송은 이 단계에서 구현하지 않는다.
* 수동 발송완료 처리를 실제 메일 발송 성공과 같은 `Method = Smtp`로 기록하지 않는다.
* 이미 차단된 대상은 기본적으로 처리 대상에 포함하지 않는다.

## 요구사항

* 공고일자 정렬 조건을 적용한다.
* 발송 건수 조건을 적용한다.
* `이미 발송된 이메일 제외` 조건을 적용한다.
* `차단 목록 제외` 조건을 적용한다.
* 조회 결과를 DataGridView에 표시한다.
* 체크박스로 대상 선택/해제할 수 있다.
* 선택 건수를 표시한다.
* 전체 선택/선택 해제 버튼을 동작시킨다.
* `발송완료 처리` 버튼을 동작시킨다.
* 발송완료 처리 전 MessageBox 확인을 표시한다.
* Supabase `SendHistory`에 다음 값으로 기록한다.

```text
Status = ManuallyMarkedSent
Method = Manual
Memo = 사용자가 발송 없이 발송완료 처리
ProcessedAt = 현재 시각
```

* 처리 후 조회 결과를 갱신한다.
* 처리 로그를 남긴다.

## 완료 조건

* Supabase 대상자를 조회할 수 있다.
* 선택 대상 수가 정확히 표시된다.
* 선택 대상에 수동 발송완료 이력이 기록된다.
* 수동 발송완료 처리 후 `이미 발송된 이메일 제외` 조건에서 대상이 제외된다.
* 컴파일 경고/에러 없이 빌드된다.

## 테스트 방법

* .NET Framework용 MSBuild로 빌드
* 업로드된 Supabase 샘플 데이터에서 대상 조회
* 일부 대상 선택 후 발송완료 처리
* `SendHistory` 기록 확인
* 재조회 시 제외 여부 확인

## 결과

* `MailSender_v2` 메일 탭의 발송 대상 조회 조건을 실제 Supabase 조회 흐름에 연결했다.
* `Recipients` 조회 결과를 DataGridView에 표시하고 체크박스 선택/전체 선택/선택 해제를 동작시켰다.
* 선택 대상 수 표시를 실제 체크 상태 기준으로 갱신하도록 구현했다.
* `발송완료 처리` 버튼을 추가 동작시켜 MessageBox 확인 후 `SendHistory`에 다음 이력을 기록하도록 했다.

```text
Status = ManuallyMarkedSent
Method = Manual
Memo = 사용자가 발송 없이 발송완료 처리
```

* 처리 후 `이미 발송된 이메일 제외` 조건에 반영되도록 재조회 흐름을 연결했다.
* .NET Framework용 MSBuild Debug 빌드 성공을 확인했다.
