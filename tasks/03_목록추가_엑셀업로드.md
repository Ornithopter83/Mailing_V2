# 작업명

목록추가_엑셀업로드

## 목표

* `목록 추가` 탭에서 Excel 파일을 선택하고 Supabase DB에 반영하는 흐름을 구현한다.
* 업로드 상세 결과와 최근 작업 로그를 화면에 표시한다.

## 배경

* 운영자는 비정기적으로 생성되는 `입찰공고_메일링리스트` 계열 Excel 파일을 업로드한다.
* 업로드 단계에서 신규, 기존, 이메일 없음, 형식 오류, 차단 목록, 중복, 기타 오류를 구분해야 한다.

## 관련 파일

* `MailSender_v2/MainForm.cs`
* `MailSender_v2/MailSender_v2.csproj`
* `MailSender_v2_revised.md`
* `tasks/목록추가.png`
* `tasks/02_DB기반구성.md`

## 변경 금지

* 실제 SMTP 발송 로직은 이 단계에서 구현하지 않는다.
* Supabase 테이블 구조는 `02_DB기반구성` 결과를 임의로 깨지 않는다.
* 업로드 중 실제 운영 파일을 수정하지 않는다.

## 요구사항

* 파일 선택 버튼으로 `.xlsx` 파일을 선택한다.
* Excel 첫 번째 시트 또는 유효한 메일링 시트를 읽는다.
* 이메일 컬럼을 탐색한다.
* 이메일을 정규화한다.
* 빈 이메일, 형식 오류, 파일 내부 중복을 분류한다.
* 차단 목록 이메일을 제외한다.
* 신규 이메일은 Supabase `Recipients`에 추가한다.
* 기존 이메일은 기관명, 공고일자, 담당자 등 가능한 정보를 갱신한다.
* Supabase `UploadHistory`에 업로드 요약을 저장한다.
* 업로드 상세 결과 DataGridView를 갱신한다.
* 최근 작업 로그를 화면에 표시한다.
* 결과 텍스트 저장 기능을 구현한다.
* 가능하면 결과 Excel 저장의 `처리요약` 시트를 구현한다.

## 완료 조건

* 샘플 Excel 파일을 선택해 업로드할 수 있다.
* 처리 결과 건수가 화면에 표시된다.
* 신규/기존/제외/오류 분류가 Supabase와 화면에 반영된다.
* 업로드 로그가 표시된다.
* 컴파일 경고/에러 없이 빌드된다.

## 테스트 방법

* .NET Framework용 MSBuild로 빌드
* 샘플 Excel 업로드
* 업로드 결과 DataGridView 확인
* Supabase의 `Recipients`, `UploadHistory` 데이터 확인
* 결과 텍스트 저장 확인

## 결과

* `목록 추가` 탭의 파일 선택 버튼을 실제 `.xlsx` 선택 대화상자와 연결했다.
* `업로드 및 처리 시작` 버튼을 Excel 처리 흐름과 연결했다.
* Excel 업로드 처리 계층을 추가했다.
  * `ExcelUploadProcessor`
  * `UploadProcessResult`
  * `UploadSummaryRow`
  * `RecipientUpsertDto`
  * `UploadHistoryDto`
* Excel 첫 번째 유효 시트에서 이메일 컬럼을 탐색하고, 다음 컬럼을 가능한 범위에서 매핑하도록 했다.
  * 이메일
  * 기관명
  * 공고일자
  * 공고명
  * 담당자
  * 연락처
* 이메일 정규화와 형식 검사를 구현했다.
* 빈 이메일, 형식 오류, 파일 내부 중복, 차단 목록, 신규/기존 항목을 분류하도록 했다.
* Supabase 설정이 있는 경우 다음 작업을 수행하도록 연결했다.
  * `Recipients`의 기존 `NormalizedEmail` 조회
  * `BlockedEmails`의 차단 `NormalizedEmail` 조회
  * `Recipients` upsert
  * `UploadHistory` insert
* Supabase 설정이 없는 경우 DB 반영 없이 파일 처리 결과만 화면에 표시하도록 했다.
* 업로드 정보 영역과 업로드 상세 결과 DataGridView가 실제 처리 결과로 갱신되도록 했다.
* 최근 작업 로그에 업로드 진행 상태를 표시하도록 했다.
* 결과 텍스트 저장 기능을 구현했다.
* 결과 Excel 저장 기능을 구현했다.
  * 1차 구현 범위: `처리요약` 시트
* .NET Framework용 MSBuild로 빌드 성공을 확인했다.

```text
MailSender_v2 -> C:\Projects\VS\EMailService\MailSender_v2\bin\Debug\MailSender_v2.exe
```

* 실제 Supabase URL/anon key가 제공되지 않아 라이브 `Recipients` / `UploadHistory` 반영 검증은 수행하지 않았다.
