# 작업명

프로젝트복사\_기반정리

## 목표

* MailSender\_v2\_revised.md를 참조하여 화면과 기능 구현

## 배경

* GUI형 개발

## 관련 파일

* 메일작성.png, 목록추가.png

## 변경 금지

* 폴더 내 전체 수정 허용

## 요구사항

* 최대한 관련파일 폼과 유사하게 만들 것.

## 완료 조건

* 경고/에러 없이 컴파일 성공

## 테스트 방법

* 직접

## 결과

* `NaraEmailSender`를 `MailSender_v2` 작업용 폴더로 복사했다.
* 솔루션/프로젝트 파일명을 `MailSender_v2.sln`, `MailSender_v2.csproj`로 변경했다.
* 프로젝트 `RootNamespace`와 `AssemblyName`, 어셈블리 표시명을 `MailSender_v2`로 정리했다.
* 기존 콘솔 진입점을 WinForms 진입점으로 교체했다.
* `MainForm.cs`를 추가하여 첨부 이미지와 문서 초안 기준의 2탭 UI 초안을 구성했다.
  * `1. 목록 추가 (입찰공고 파일 업로드)`
  * `2. 발송 대상 / 메일 작성`
* 복사 과정에서 따라온 `MailSender_v2\.vs`, `MailSender_v2\obj` 캐시 폴더를 정리했다.
* `dotnet build`는 .NET Framework ClickOnce 관련 `GenerateTrustInfo` 작업 미지원으로 실패했다.
* .NET Framework용 MSBuild로 빌드 확인 완료:

```text
C:\Program Files\Microsoft Visual Studio\2022\Professional\MSBuild\Current\Bin\MSBuild.exe C:\Projects\VS\EMailService\MailSender_v2\MailSender_v2.csproj /p:Configuration=Debug /p:Platform=AnyCPU /v:minimal
```

```text
MailSender_v2 -> C:\Projects\VS\EMailService\MailSender_v2\bin\Debug\MailSender_v2.exe
```
