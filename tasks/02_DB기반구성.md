# 작업명

DB기반구성

## 목표

* `MailSender_v2`에서 사용할 Supabase DB 기반을 구성한다.
* Supabase 접속 설정, 기본 테이블 접근 확인, 데이터 접근 계층의 최소 구조를 만든다.

## 배경

* v2는 Excel에서 이메일을 즉시 발송하는 방식이 아니라 DB에 대상 정보를 저장한 뒤 조회/발송/발송완료 처리하는 흐름이다.
* 수동 발송완료 처리와 이미 발송된 이메일 제외 조건을 안정적으로 처리하려면 Supabase의 `SendHistory` 기반 이력이 필요하다.

## 관련 파일

* `MailSender_v2/MailSender_v2.csproj`
* `MailSender_v2/App.config`
* `MailSender_v2/MainForm.cs`
* `MailSender_v2_revised.md`
* `CurrentWork.md`

## 변경 금지

* `NaraInfoCollectorBot` 프로젝트는 수정하지 않는다.
* SMTP 실제 발송 로직은 이 단계에서 구현하지 않는다.
* UI 세부 동작 연결은 DB 초기화 확인에 필요한 최소 범위만 수정한다.

## 요구사항

* Supabase 접속 방식을 결정한다.
  * 1차 후보: `HttpClient` 기반 PostgREST REST API 직접 호출
  * 2차 후보: .NET Framework 4.8에서 호환 가능한 Supabase/PostgREST 클라이언트 패키지
* Supabase URL과 anon key 설정 구조를 정한다.
* service role key는 데스크톱 앱, 코드, 문서, 로그에 기록하지 않는다.
* Supabase RLS 정책을 전제로 하며, 데스크톱 앱에서는 anon key만 사용한다.
* 다음 테이블을 생성한다.
  * `Recipients`
  * `UploadHistory`
  * `SendHistory`
  * `BlockedEmails`
* 테이블 생성은 Supabase SQL Editor 또는 별도 마이그레이션 SQL 기준으로 관리한다.
* 앱 시작 시 Supabase 연결 상태와 기본 테이블 접근 가능 여부를 확인한다.
* DB 연결/확인 실패 시 사용자에게 확인 가능한 오류를 남긴다.
* 민감 정보는 DB나 로그에 기록하지 않는다.

## 완료 조건

* 앱 실행 시 Supabase 연결 설정을 읽는다.
* 4개 기본 테이블 접근 가능 여부를 확인한다.
* 빈 DB 상태에서도 앱이 오류 없이 시작된다.
* 컴파일 경고/에러 없이 빌드된다.

## 테스트 방법

* .NET Framework용 MSBuild로 `MailSender_v2.csproj` 빌드
* 앱 실행 후 Supabase 연결 설정 로드 확인
* Supabase 기본 테이블 접근 여부 확인

## 결과

* Supabase 접속 방식은 1차 구현 기준으로 `HttpClient` 기반 PostgREST REST API 직접 호출로 결정했다.
* `config.json`에서 Supabase 설정을 읽는 `AppSettings` / `SupabaseSettings`를 추가했다.
  * `Supabase.Url`
  * `Supabase.AnonKey`
* 기존 SMTP 설정 필드와 충돌하지 않도록 기존 `config.json`의 `SmtpUser`, `SmtpPw`, `Subject`, `SendInterval`, `MaxCount` 필드를 보존 가능한 설정 모델로 구성했다.
* Supabase `service_role` key는 앱 설정 구조에 포함하지 않았다.
* `Data` 계층의 최소 구조를 추가했다.
  * `SupabaseRestClient`
  * `DatabaseInitializer`
  * `SupabaseConnectionCheckResult`
  * `SupabaseTableCheckResult`
* 앱 시작 시 `Supabase.Url` / `Supabase.AnonKey`가 있으면 다음 테이블에 대해 PostgREST `limit=0` 접근 확인을 수행하도록 연결했다.
  * `Recipients`
  * `UploadHistory`
  * `SendHistory`
  * `BlockedEmails`
* Supabase 설정이 비어 있으면 앱은 오류 없이 시작하고, 상태바/업로드 로그에 설정 필요 메시지를 표시한다.
* Supabase SQL Editor에서 사용할 수 있는 스키마 초안 파일을 추가했다.
  * `MailSender_v2/Database/supabase_schema.sql`
* 실제 Supabase URL/anon key가 제공되지 않았으므로 라이브 테이블 접근은 수행하지 않았다.
* .NET Framework용 MSBuild로 빌드 성공을 확인했다.

```text
MailSender_v2 -> C:\Projects\VS\EMailService\MailSender_v2\bin\Debug\MailSender_v2.exe
```
