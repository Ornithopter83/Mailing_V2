# 작업명

발송 진행 상태 팝업

## 목표

* SMTP 발송을 수행하면 발송 진행 상태 팝업을 표시한다.
* 팝업에는 현재 메일을 보내는 중인 대상 이메일을 표시한다.
* 팝업에는 각 대상의 발송 완료 여부를 표시한다.
* 발송 완료 후 사용자가 전체 성공/실패 상태를 한눈에 확인할 수 있게 한다.

## 배경

* 현재 발송은 로그 TextBox와 완료 MessageBox 중심으로 상태를 확인한다.
* 여러 대상에게 순차 발송할 때 사용자는 현재 어떤 이메일을 발송 중인지, 어느 대상이 완료되었는지 즉시 보기 어렵다.
* 발송 간격이 있고 첨부/이미지 포함 메일은 시간이 걸릴 수 있으므로 진행 상태 전용 팝업이 필요하다.

## 관련 파일

* `MailSender_v2/MainForm.cs`
* `MailSender_v2/Mailing/MailSendService.cs`
* `MailSender_v2/Mailing/MailSendResult.cs`
* `MailSender_v2/Mailing/RecipientListItem.cs`
* `CurrentWork.md`

## 변경 금지

* SMTP 발송 순서와 발송 간격 정책을 임의로 변경하지 않는다.
* DB 조회 대상과 수동 입력 대상의 발송 순서를 변경하지 않는다.
* 수동 입력 대상만 발송하는 경우의 동작 의미를 변경하지 않는다.
* 발송 중인 실제 이메일 주소를 파일 로그나 문서에 불필요하게 남기지 않는다.
* 발송 완료 후 자동으로 다음 DB 목록을 재조회하지 않는다.

## 요구사항

### 1. 팝업 표시 시점

* 사용자가 `발송하기`를 누르고 최종 확인 MessageBox에서 `Yes`를 선택하면 발송 진행 상태 팝업을 표시한다.
* SMTP 발송을 시작하기 전에 전체 발송 대상 목록을 팝업에 표시한다.
* 발송 대상이 없거나 유효성 검사 실패 시에는 팝업을 표시하지 않는다.

### 2. 팝업 표시 항목

* 팝업에는 다음 정보를 표시한다.
  * 전체 대상 수
  * 완료 수
  * 성공 수
  * 실패 수
  * 현재 발송 중인 이메일
  * 대상별 상태
* 대상별 상태는 최소 다음 값을 사용한다.
  * 대기
  * 발송 중
  * 성공
  * 실패

### 3. 대상 구분

* 대상 목록에는 DB 조회 대상과 수동 입력 대상을 함께 표시한다.
* 대상 구분 컬럼을 두어 `DB`, `수동`을 구분한다.
* 중복 제거 후 실제 발송 대상 기준으로 표시한다.

### 4. UI 동작

* 발송 중에는 팝업을 사용자가 닫지 못하게 하거나, 닫기 시 발송은 계속 진행된다는 안내를 표시한다.
* 1차 구현에서는 발송 취소 기능은 포함하지 않는다.
* 발송 완료 후에는 팝업을 닫을 수 있게 한다.
* 팝업은 메인 폼 아이콘과 동일한 아이콘을 사용한다.

### 5. 발송 상태 갱신

* 각 대상 발송 시작 시 해당 row를 `발송 중`으로 변경한다.
* 발송 성공 시 `성공`으로 변경한다.
* 발송 실패 시 `실패`로 변경하고 오류 메시지를 표시한다.
* 완료 수, 성공 수, 실패 수는 발송 결과에 따라 즉시 갱신한다.
* 현재 발송 중인 이메일 표시도 매 대상마다 갱신한다.

### 6. 발송 완료 후 처리

* 기존처럼 `SendHistory` 기록, 텍스트 보고서 생성, SMTP 사용자 보고 메일 전송을 수행한다.
* 발송 완료 후 선택된 대상 목록은 기존 정책대로 화면에서 초기화한다.
* 발송 완료 후 DB 목록 자동 재조회는 하지 않는다.

## 완료 조건

* SMTP 발송 시작 시 진행 상태 팝업이 표시된다.
* 팝업에 전체 대상, 현재 발송 중인 이메일, 대상별 상태가 표시된다.
* 각 대상 발송 시작/성공/실패 시 UI가 갱신된다.
* 발송 완료 후 성공/실패 건수를 확인할 수 있다.
* 발송 완료 후 팝업을 닫을 수 있다.
* 빌드가 성공한다.

## 테스트 방법

* .NET Framework용 MSBuild로 빌드한다.
* DB 조회 대상 2건 이상을 선택하고 발송 시 팝업이 표시되는지 확인한다.
* 발송 중인 이메일 표시가 순차적으로 바뀌는지 확인한다.
* 성공/실패 상태가 대상별로 갱신되는지 확인한다.
* 수동 입력 대상이 있는 경우 `수동` 구분으로 표시되는지 확인한다.
* 발송 완료 후 기존 대상 목록이 초기화되고 자동 재조회되지 않는지 확인한다.

## 결과

* SMTP 발송 시작 전 `MailSendProgressForm` 팝업을 표시하도록 구현했다.
* 팝업에는 전체 대상 수, 완료 수, 성공 수, 실패 수, 현재 상태/현재 발송 이메일을 표시한다.
* 대상 목록은 DB 조회 대상과 수동 입력 대상을 `DB`, `수동` 구분으로 나누어 표시한다.
* `MailSendService.SendAsync`에 대상별 진행 콜백을 추가하여 발송 시작 시 `발송 중`, 성공 시 `성공`, 실패 시 `실패`로 row 상태를 갱신한다.
* 발송 중에는 팝업을 닫을 수 없고, 발송 완료 또는 오류 후에는 닫을 수 있게 했다.
* 발송 이력 기록, 텍스트 보고서 생성, SMTP 사용자 보고 메일 전송, 대상 목록 초기화 흐름은 기존 정책을 유지했다.
* `MainForm.cs`는 코드 기반 UI이므로 프로젝트 파일에서 `SubType`을 `Code`로 유지하도록 보정했다.
* MSBuild 검증은 시도했으나, 샌드박스가 `C:\Users\SUHO_DEV_PC\AppData\Local\Microsoft SDKs` 접근을 차단했고 권한 승인도 사용량 제한으로 거절되어 완료하지 못했다.
