# 작업명

발송 결과를 SMTP 사용자에게 보고

## 목표

* DB 조회 대상에게 SMTP 메일을 1건 이상 발송한 경우, 발송 결과를 SMTP 사용자에게 보고 메일로 전송한다.
* 보고 메일에는 대상 이메일과 발송 성공 여부를 포함한다.
* 보고 메일에는 실제 발송에 사용한 이미지 포함 본문을 함께 포함한다.

## 배경

* 운영자는 발송 완료 후 별도 로그 창이나 텍스트 보고서를 열지 않고도 SMTP 계정 메일함에서 발송 결과를 확인할 필요가 있다.
* 수동 입력 대상만 발송한 경우는 DB 목록 발송 보고 목적과 다르므로 보고 메일 대상에서 제외한다.
* 보고 메일은 실제 발송 이력 관리를 위한 메일이 아니라 운영자 확인용이다.

## 관련 파일

* `MailSender_v2/MainForm.cs`
* `MailSender_v2/Mailing/MailSendService.cs`
* `MailSender_v2/Mailing/MailSendResult.cs`
* `CurrentWork.md`

## 변경 금지

* 보고 메일을 `SendHistory`에 발송 이력으로 기록하지 않는다.
* 수동 입력 대상만 발송한 경우에는 보고 메일을 보내지 않는다.
* SMTP 계정, 비밀번호, 토큰 등 민감 정보를 본문이나 로그에 남기지 않는다.
* 실제 수신자 목록 외 불필요한 개인정보를 보고 메일에 포함하지 않는다.

## 요구사항

### 1. 보고 메일 발송 조건

* DB 조회 대상에게 SMTP 발송을 1건 이상 시도한 경우에만 보고 메일을 보낸다.
* DB 조회 대상 0건, 수동 입력 대상만 있는 경우에는 보고 메일을 보내지 않는다.
* DB 조회 대상 발송 결과에 성공/실패가 섞여 있어도 보고 메일을 보낸다.

### 2. 보고 메일 수신자

* 보고 메일 수신자는 `config.json`의 `SmtpUser` 값으로 한다.
* 발신자도 기존 SMTP 발송과 동일하게 `SmtpUser`를 사용한다.

### 3. 보고 메일 본문

* 보고 메일 제목은 원본 메일 제목을 포함한다.
* 본문에는 다음 내용을 포함한다.
  * 전체 발송 시도 건수
  * 성공 건수
  * 실패 건수
  * 대상 이메일
  * 발송 성공 여부
  * 발송 메시지
  * 이미지 포함 발송본문
* 이미지 포함 발송본문은 기존 `{ID}` 토큰과 `Images` 설정을 사용해 CID 이미지로 표시한다.
* 원본 메일의 첨부 파일도 보고 메일에 동일하게 첨부한다.

### 4. 로그

* 보고 메일 발송 시작과 완료를 SMTP 로그에 남긴다.
* 보고 메일 발송 실패 시 기존 SMTP 오류 처리 흐름에서 원인을 표시한다.

## 완료 조건

* DB 조회 대상 발송 결과가 1건 이상이면 SMTP 사용자에게 보고 메일이 전송된다.
* 보고 메일에 대상 이메일과 발송 성공 여부가 표 형태로 표시된다.
* 보고 메일에 이미지 포함 발송본문이 표시된다.
* 보고 메일에 원본 메일 첨부 파일이 포함된다.
* 수동 입력 대상만 발송한 경우에는 보고 메일이 전송되지 않는다.
* 빌드가 성공한다.

## 테스트 방법

* .NET Framework용 MSBuild로 빌드한다.
* DB 조회 대상 1건 이상을 선택하고 SMTP 발송 후 `SmtpUser` 메일함에 보고 메일이 도착하는지 확인한다.
* 보고 메일에서 대상 이메일, 성공/실패 여부, 이미지 포함 본문, 첨부 파일을 확인한다.
* 수동 입력 대상만 발송했을 때 보고 메일이 오지 않는지 확인한다.

## 결과

* `MailSendService`에 SMTP 사용자에게 발송 결과 보고 메일을 보내는 기능을 추가했다.
* `MainForm.SendMailAsync`에서 DB 조회 대상 발송 결과가 1건 이상 있을 때만 보고 메일을 보내도록 했다.
* 보고 메일은 `SmtpUser`가 자신에게 받는 형태이며, 대상 이메일/발송성공여부/메시지 표와 이미지 포함 발송본문을 포함한다.
* 보고 메일도 원본 메일과 동일한 첨부 파일을 포함하도록 보정했다.
* 보고 메일은 `SendHistory`에 별도 이력으로 기록하지 않는다.
